# WebEcho AI — 项目交接文档

## 一、项目概况

**项目名称**：WebEcho AI 克隆平台  
**交接日期**：2026-02-26  
**最后更新**：2026-02-27（平台管理后台监督脚本、测试监督、完整克隆 E2E、安全审计、漏洞修复、性能优化、定价 traffic 修复）  
**技术栈**：Next.js 16 + TypeScript + Tailwind CSS 4

**核心文档**：`PROJECT.md` 含完整项目描述，开发时以该文档为最高优先级参考。

**主要依赖**：playwright、@octokit/rest、simple-git、extract-zip、archiver、stripe、@supabase/ssr、@anthropic-ai/sdk、resend、pg、bcryptjs、otplib、qrcode

**迁移文件**：`supabase/migrations/` 下按时间戳排序；推荐+API Key 可单独执行 `run-manual.sql`

## 八、注意事项

- **任务存储**：配置 `SUPABASE_SERVICE_ROLE_KEY` 后使用 Supabase `clone_tasks` 表持久化。需先执行迁移创建表
- **profiles 表**：退款额度回退需 profiles 表。新建用户通过触发器自动创建 profile。推荐计划需 referral_code、referred_by 字段，执行迁移 20260226500001_referral.sql
- **企业版 API**：执行迁移 20260226600001_api_keys.sql 创建 api_keys 表。克隆接口支持 `Authorization: Bearer we_xxx`，与 Session 二选一。若未配置 DATABASE_URL，可复制 `supabase/migrations/run-manual.sql` 到 Supabase SQL Editor 手动执行
- **执行迁移**：在 `.env.local` 配置 `DATABASE_URL`（Supabase Dashboard -> Database -> Connection string）后运行 `npm run db:migrate`
- **hosted_sites**：托管部署需执行 `20260226100001_create_hosted_sites.sql` 创建 hosted_sites 表
- **下载时效**：未配置 R2 时 zip 存于 `/tmp`，可能被系统清理；生产部署及托管部署均需配置 R2
- **Supabase 配置**：登录/注册依赖 Supabase 项目，需在控制台启用 Email Auth
- **Stripe / 额度**：配置 Stripe 后新建克隆走支付流程；未配置时使用 profiles.credits，额度不足无法创建，detect-complexity 返回 canProceed
- **Docker 测试**：本机已安装 Docker 时，克隆流程会自动运行构建验证并影响 qualityScore
- **内嵌浏览器**：需 Playwright，首次运行会下载 Chromium
- **克隆预览**：preview-server 需本机或长驻进程环境（非 Vercel serverless），R2 或本地 zip 至少配置其一
- **平台管理后台**：执行 `npm run db:migrate` 后，运行 `npm run seed:admin -- admin@webecho.ai <密码>` 创建首任管理员，访问 `/platform-admin/login` 登录
- **维护模式**：平台管理后台「系统配置」中开启后，`/api/clone/create` 返回 503，暂停新任务接入
- **APP 克隆**：截图模式 3 额度、APK 模式 8 额度、流量模式 20 额度；暂仅支持额度支付（非 Stripe）；截图：screenshots 为 base64/data URL 数组，建议 1–2 张；APK/流量：先调用 /api/app/upload-apk 上传，create 时传 appR2Key；流量模式需构建 `docker build -f docker/Dockerfile.android -t webecho-android .`；需配置 R2 与 apktool（APKTOOL_PATH）；结果为 Expo 项目，解压后 `npm install && npx expo start` 用 Expo Go 预览
- **价格调控**：平台管理后台「定价控制台」可设每项固定价格或成本倍数，保存后首页、定价页、clone/new、detect-complexity 即时生效，无需重启
- **配置操作日志**：pricing、system、signatures 每次修改写入 platform_config_logs，不可删除
- **定价历史回滚**：每次保存定价写入 pricing_history 快照，定价控制台支持回滚到任意历史
- **TOTP 双因素**：管理员在 /platform-admin/security 启用后，登录需输入 Authenticator 动态码
- **监控告警**：配置 RESEND_API_KEY 与 ALERT_EMAILS（或 platform_admins）后，克隆失败、Stripe webhook 异常会邮件通知；系统配置可开启「失败率过高自动维护」；/platform-admin/monitoring 查看成功率、健康、慢请求
- **安全加固**：lib/url-validate 在 clone/create、detect-complexity、browser/session、browser/action(navigate)、stripe/checkout、clone-worker 校验 URL；平台 Session 使用 HMAC 签名；status/download 归属用户时需鉴权；deploy/preview-session 校验任务归属；建议生产环境单独配置 PLATFORM_ADMIN_SESSION_SECRET
- **测试监督**：`npm run test:supervision` 需先 `npm run dev`；`npm run test:admin-supervision` 需先创建管理员或设 `ADMIN_AUTO_SEED=1`；`ADMIN_USE_FETCH=1` 可避免 Playwright 依赖；完整 E2E 需 Supabase 配置，未配置 Stripe 时使用额度模式；`npx playwright install chromium` 可安装浏览器以启用完整 E2E
